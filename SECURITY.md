# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x.x   | ✅        |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Please report security issues by emailing: **security@example.com** (replace with your actual address)

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will acknowledge within **48 hours** and aim to release a fix within **7 days** for critical issues.

## Scope

- Data privacy: books and reading progress are stored locally only
- No network access to external servers
- File access limited to user-selected documents via Android's Storage Access Framework
- No analytics or telemetry

## Out of Scope

- Issues in third-party dependencies (report upstream)
- Issues requiring physical device access
